# 论文润色平台设计

日期：2026-05-02

## 产品边界

本产品是一个免费、公开访问、无需登录的论文润色平台，目标是帮助用户改进学术表达、提升文本自然度、保护论文格式，并提供可追溯的修改记录。

产品不承诺绕过或针对任何第三方检测平台，也不保证任何外部 AI 检测或查重系统的具体分数。系统只围绕可验证的写作质量和格式保护指标设计，例如原意一致、术语保留、引用完整、格式不变、长度控制、修改可审阅。

## 核心决策

- 平台公开访问，任何人拿到网址都可以使用。
- 不登录，不让用户提供模型 API Key，不做付费计划，不在前台展示使用额度。
- 平台统一使用自有模型配置，可通过模型适配层接入 GPT-5.5。
- 任务通过随机取件码归档。
- 一个取件码就是一个可持续使用的任务空间。用户输入有效取件码后，可以继续在该码下新增论文任务。
- 浏览器会记住当前取件码，直到用户主动退出取件码。
- 取件码及其关联数据保存 7 天。过期或不存在的取件码不会复用。
- 用户可以在 7 天内手动删除单个任务，或删除该取件码下的全部数据。
- 第一版以 `.docx` 为核心。上传 `.doc` 时，系统自动转换为 `.docx`，再进入统一 DOCX 处理链。
- 前端不设置可见使用限制。后端仍然需要队列、超时恢复、Worker 隔离和稳定性调度，避免任务互相阻塞。

## 推荐方案

采用 DOCX 优先、保守回写、异步处理的论文润色平台方案。

主流程如下：

1. 上传 `.docx` 或 `.doc` 文件。
2. 如为 `.doc`，先转换为 `.docx`。
3. 解析 DOCX 结构，生成树形大纲。
4. 使用大模型辅助校验段落分类和大纲结构。
5. 用户审阅大纲和段落选择。
6. 对选中的段落进入质量优先的多候选润色流程。
7. 对每个润色结果做确定性校验。
8. 导出润色后的 DOCX、修改记录报告和段落对照表。

## 用户流程

第一屏是任务入口，不做登录。用户可以创建新取件码，也可以输入已有取件码进入任务空间。用户进入有效取件码后，浏览器会保持该状态；刷新页面或关闭浏览器后再次打开，仍会自动进入当前任务空间。用户点击“退出取件码”后，本地状态才会清除。

在一个取件码任务空间里，用户可以持续上传多篇论文。每次批量上传会创建一批任务，但这些任务都挂在同一个取件码下。

单篇论文流程包括：

- 上传和格式转换
- 文档结构解析
- 用户审阅
- 润色队列
- 结果审阅
- 文件导出

界面采用混合式布局。上传和进度展示采用向导式流程；大纲审阅和结果对照采用工作台式界面。桌面端左侧显示树形大纲，右侧显示段落详情；手机端折叠为上下结构。

## 文档结构解析

解析流程分三层。

第一层是确定性 DOCX 解析。系统读取段落样式、标题级别、多级编号、章节顺序，以及摘要、关键词、正文、参考文献、附录、致谢、表格、图片、题注、脚注、尾注等常见论文结构。

第二层是大模型辅助分类。对于规则难以判断的片段，模型会辅助判断其类型，例如摘要正文、正文自然段、标题、小标题、图表题注、参考文献条目、表格文本、附录内容、高风险段落等，并补充分类理由。

第三层是兜底安全校验。凡是包含表格、脚注、尾注、图表题注、参考文献、目录、标题、小标题、复杂域、密集交叉引用、批注、修订痕迹等格式风险的内容，默认标记为跳过。

树形大纲显示章节、小节、可处理段落和跳过段落。跳过节点也必须可见，并显示跳过原因。每个段落节点可以展示状态、风险等级、长度、引用数量、编号检测结果和模型分类置信度。

默认处理规则：

- 正文自然段和摘要正文默认可选。
- 标题、摘要标题、关键词标题、目录、参考文献、图表题注、附录和复杂格式段落默认跳过。
- 用户可以手动覆盖系统分类。
- 如果用户把高风险跳过段落改为需要润色，界面必须要求二次确认格式风险。

摘要正文润色完成后，系统会从润色后的摘要中提取关键词建议。关键词不会自动替换，必须由用户确认后才写入最终文件。

## 润色流程

润色流程采用质量优先策略。

进入模型润色前，系统先提取并冻结受保护元素：

- `（1）`、`1）`、`①` 等编号前缀
- 参考文献引用标记
- 上标引用
- 交叉引用占位
- 英文缩写
- 专有名词
- 专业术语
- 数字
- 类公式片段

这些受保护元素不能被删除、增加、错误改写或错误移动。

每个选中段落生成 2-3 个候选润色版本。候选版本必须保持原意，保持学术语气，减少模板化和过度工整的表达，保留受保护元素，并把长度控制在原文约 ±5% 以内。

复核模型会从候选版本中选择最优结果，评分维度包括：

- 语义一致性
- 学术表达自然度
- 模板化表达降低程度
- 引用完整性
- 编号完整性
- 受保护术语保留情况
- 长度偏差
- 整体可读性

如果校验失败，系统采用诊断式重试，而不是盲目重复同一提示词。系统先判断失败原因，再动态调整提示词、规则约束或候选选择策略。

常见失败原因包括：

- 长度超出容忍范围
- 引用缺失或被改写
- 编号前缀缺失或被改写
- 受保护术语缺失
- 语义偏移
- 表达过于口语化
- 格式风险过高
- 模型输出结构异常

每个段落最多进行 5 轮诊断式重试。仍然失败时，该段落进入“需要人工决定”状态，用户可以选择继续重试或跳过该段。

## DOCX 格式保护

第一版采用极保守 DOCX 回写策略。系统不重建整篇文档，而是在原 DOCX 包结构上做最小文本替换，尽量保留原始样式、关系文件、页眉页脚、图片、表格、域代码和版式关键结构。

第一版默认只处理纯正文自然段和摘要正文。表格、脚注、尾注、题注、目录、参考文献、批注、修订痕迹、复杂域、密集交叉引用以及图表附近题注等内容默认跳过。

对选中段落，系统只替换可控文本运行。编号、引用标记、上标引用和交叉引用占位不会作为可自由改写文本发送给模型。

上传 `.doc` 文件时，系统先转换为 `.docx` 再解析。界面应提示用户：旧版 `.doc` 格式转换可能带来少量版式差异。

## 导出内容

每个完成任务提供三类导出文件：

- 润色后的 `.docx`
- 修改记录报告
- 段落对照表

修改记录报告包括任务信息、已处理段落数、跳过段落数、失败段落数、重试次数、引用校验、长度变化、关键词建议和风险提示。

段落对照表包括大纲路径、段落类型、原文、润色后文本、处理状态、跳过原因、长度变化、重试次数和校验结果。

## 后端状态模型

核心对象包括：

- 取件码空间
- 论文任务
- 段落记录
- 导出文件

取件码空间保存随机码、创建时间、最后访问时间、过期时间和任务列表。

论文任务保存原始上传文件、转换后的 DOCX、解析结果、当前状态、进度、错误信息、导出文件引用和处理统计。

段落记录保存大纲路径、段落类型、原文、润色后文本、是否选中、跳过原因、风险等级、引用校验结果、编号校验结果、术语校验结果、重试次数和校验状态。

导出文件保存最终 DOCX、修改记录报告、段落对照表和过期时间。

任务状态包括：

- `uploaded`
- `converting`
- `parsing`
- `awaiting_review`
- `queued_rewrite`
- `rewriting`
- `awaiting_manual_decision`
- `exporting`
- `completed`
- `failed`
- `deleted`
- `expired`

段落状态包括：

- `skipped`
- `selected`
- `queued`
- `rewriting`
- `validated`
- `retrying`
- `needs_manual_decision`
- `failed`

Worker 划分：

- 转换 Worker：负责 `.doc` 转 `.docx`
- 解析 Worker：负责 DOCX 结构解析、大纲生成和模型辅助分类
- 润色 Worker：负责候选生成、复核、诊断式重试和段落校验
- 导出 Worker：负责 DOCX 回写、报告生成和段落对照表生成

调度器负责队列管理、进度聚合、超时恢复、Worker 隔离、失败状态流转和清理任务调度。

## 保存与删除

取件码空间和所有关联文件默认保存 7 天。

用户手动删除后，数据立即标记为 `deleted`，并进入异步文件清理队列。

定时清理任务会删除过期或已删除的数据记录和对象存储文件，包括原始文件、转换文件、导出 DOCX、报告和段落对照表。

## 异常处理

任务级异常包括文件损坏、`.doc` 转换失败、DOCX 解析失败、导出失败和存储失败。任务级异常会让论文任务进入 `failed` 或 `awaiting_manual_decision`，并展示明确原因。

段落级异常包括长度超限、引用丢失、编号变化、术语缺失、语义偏移、复杂格式风险和模型输出异常。段落级异常会先触发最多 5 轮诊断式重试，仍失败后进入人工决定状态。

界面应允许用户使用部分完成结果。单个段落失败不应导致整篇论文不可导出。系统必须清楚展示哪些段落已完成、哪些被跳过、哪些需要确认、哪些失败。

## 质量校验

回写前必须执行确定性校验：

- 编号前缀不变
- 引用标记不变
- 上标和交叉引用占位保留
- 受保护术语保留
- 关键数字保留
- 长度控制在约 ±5%
- 段落类型仍允许润色
- 摘要正文被润色后已生成关键词建议

最终导出前必须校验：

- DOCX 能被 Word/WPS 正常打开
- 所有已处理段落都有对照记录
- 跳过段落没有被修改
- 报告统计与段落记录一致
- 导出文件关联到正确的取件码和任务

## 测试覆盖

测试样本应包括：

- 普通本科论文
- 含自动目录的论文
- 含脚注的论文
- 含表格的论文
- 含图题和表题的论文
- 含交叉引用的论文
- 含上标参考文献标记的论文
- 含多级编号的论文
- 旧版 `.doc` 输入样本

验收标准：

- 选中段落润色后表达自然且符合学术语体。
- 跳过段落不被修改。
- 导出的 DOCX 能被 Word/WPS 正常打开。
- 编号和引用不丢失。
- 段落对照表与导出文档一致。
- 报告统计与段落状态一致。
- 7 天内可以通过取件码找回任务。
- 过期记录会被删除。

## 第一版不做的内容

- 不保证任何第三方检测平台的具体分数。
- 不支持 PDF 输入或输出。
- 不做 `.doc` 原生编辑管线。
- 不全面编辑表格、脚注、尾注、参考文献、题注或修订痕迹。
- 不做用户账号系统。
- 不支持用户自带模型 API Key。
